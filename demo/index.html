Capstone Project Sprint 3: Group F, Pitch Grader

Audrey Kim, Andrew Renninger, Andrew Helms

Project Overview

Our goal was to to create an application for The University of Alabama's baseball team to allow pitches to be graded based on stipulations, ranges, and weights.

  • Manual grading of pitches is time-consuming.
  • Our application will enable faculty to use a standardized grading system, ensuring accurate results without the potential for human error associated with manual grading.
  • We want to also output percentages for a pitcher, advising them on what pitches to throw for their best results, based on their current arsenal

Technologies Used

NPM, Javascript, Vue.js, Bootstrap, Github Pages

Key Features

  • Web Application:
    • Pitch grader input fields: Various fields for each pitch type to allow the user to enter the data for multiple pitches
    • Pitch grader input field for handedness: Allow the user to select whether each pitch was left or right-handed before entering the input fields, as this affects the weights associated with each pitch type
    • Pitch grader option for multiple pitches: Allow the user to enter pitch details for multiple pitches before calculating grades for each
    • Pitch grader calculator: Based on given data, the application will provide a grade based on predefined weights for each pitch the user has entered
    • Pitch weights editor: A page designed to allow the user to upload a different JSON file that defines each pitch type and weights associated with them

Future Enhancements

In the future, we could utilize machine learning to calculate the optimal pitch grading ranges, as well as the recommended throw percentages for a pitcher based on their arsenal, possibly even for a certain opponent.
